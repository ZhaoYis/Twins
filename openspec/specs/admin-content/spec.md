## Requirements
### Requirement: List Generated Content

The system SHALL provide an API endpoint to list all generated content with pagination and filtering.

#### Scenario: List content with pagination
- **WHEN** admin requests `GET /api/admin/content?page=1&limit=20`
- **THEN** the system SHALL return paginated content list with:
  - id, topic, model_used, created_at
  - user email
  - content preview (first 100 chars)

#### Scenario: Filter by user
- **WHEN** admin requests `GET /api/admin/content?userId=xxx`
- **THEN** the system SHALL return content from that user only

#### Scenario: Filter by date range
- **WHEN** admin requests `GET /api/admin/content?startDate=2024-01-01&endDate=2024-12-31`
- **THEN** the system SHALL return content within the date range

#### Scenario: Filter by model
- **WHEN** admin requests `GET /api/admin/content?model=gpt-4`
- **THEN** the system SHALL return content generated by that model

### Requirement: View Content Details

The system SHALL provide an API endpoint to view full content details.

#### Scenario: Get content by ID
- **WHEN** admin requests `GET /api/admin/content/:id`
- **THEN** the system SHALL return:
  - Full content text
  - Topic
  - Model used
  - Style profile reference
  - User info
  - Creation timestamp

#### Scenario: Content not found
- **WHEN** admin requests non-existent content ID
- **THEN** the system SHALL return 404 Not Found

### Requirement: Delete Content

The system SHALL allow admins to delete generated content.

#### Scenario: Delete content
- **WHEN** admin requests `DELETE /api/admin/content/:id`
- **THEN** the system SHALL:
  - Permanently remove the content record
  - Log the action

### Requirement: Content Statistics

The system SHALL provide content generation statistics.

#### Scenario: View content stats
- **WHEN** admin requests `GET /api/admin/stats`
- **THEN** the system SHALL return:
  - Total content count
  - Content count by model
  - Content count by day/week/month
  - Average content length

### Requirement: Content Management UI

The system SHALL provide a content management page at `/admin/content`.

#### Scenario: View content list
- **WHEN** admin navigates to `/admin/content`
- **THEN** the page SHALL display a table with:
  - Topic preview
  - User email
  - Model used
  - Creation date
  - Actions (view, delete)

#### Scenario: Filter content
- **WHEN** admin applies filters
- **THEN** the content list SHALL update accordingly

#### Scenario: View content detail
- **WHEN** admin clicks on a content row
- **THEN** a modal or detail view SHALL show full content

#### Scenario: Export content data
- **WHEN** admin clicks "Export" button
- **THEN** the system SHALL download content data as JSON or CSV
