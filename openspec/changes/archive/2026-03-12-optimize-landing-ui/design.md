## Context

当前首页各组件使用了一套完整的设计系统（Minimal SaaS + Bento Grid + Gradient），但在实际使用中存在一些视觉细节问题，影响了整体的精致度和专业感。主要问题包括：

1. 装饰性元素过多（多层渐变背景、复杂的连接线）
2. 某些区域的视觉层次不够清晰
3. 间距和排版的统一性可以改进
4. 部分边框和装饰过于复杂

## Goals / Non-Goals

**Goals:**
- 简化各区域的背景装饰，减少视觉噪音
- 优化视觉层次，使信息传递更加清晰
- 统一间距和排版规范
- 保持响应式设计的兼容性
- 不改变现有的功能逻辑和用户交互流程

**Non-Goals:**
- 不修改配色方案（保持现有的渐变风格）
- 不改变布局结构（保持现有的section划分）
- 不添加新的功能或交互
- 不修改移动端的响应式布局逻辑

## Decisions

### D1: 简化背景装饰层次
**Decision:** 减少Hero和其他section的多层背景装饰
- **Rationale:** 当前多层渐变和mesh gradient叠加虽然有层次感，但容易造成视觉混乱
- **Alternative:** 保留主背景层和一到两层轻量装饰即可

### D2: 统一卡片视觉权重
**Decision:** Hero的stats区域使用统一尺寸的紧凑卡片
- **Rationale:** 当前4列等宽布局在小屏幕上可能显得拥挤，使用更紧凑的间距和统一高度可以改善
- **Alternative:** 考虑使用更简洁的横排布局

### D3: 简化连接线
**Decision:** HowItWorks步骤之间的连接线改为简洁的单色线条或完全移除
- **Rationale:** 当前渐变连接线过于复杂，与整体简洁风格不协调
- **Alternative:** 保持箭头图标作为视觉引导即可

### D4: FAQ折叠面板简化
**Decision:** 简化FAQ的边框处理，使用更轻量的分隔线
- **Rationale:** 当前每个FAQ项目都有完整的bento-card边框略显笨重
- **Alternative:** 使用简洁的列表式布局，底部用轻量分隔线

## Risks / Trade-offs

- **[风险]** 过度简化可能导致页面视觉上过于平淡
  - **→ 缓解:** 保留适当的hover效果和渐变强调元素，确保交互反馈

- **[风险]** 修改可能与现有设计系统其他页面不一致
  - **→ 缓解:** 仅修改landing页面组件，不影响其他页面

- **[风险]** 简化装饰可能影响品牌识别度
  - **→ 缓解:** 保留关键的品牌元素（logo、渐变色调、gradient-text）

## Migration Plan

1. 逐个优化每个组件文件
2. 每个组件优化后进行视觉验证
3. 确保响应式布局在各个断点正常工作
4. 测试暗色模式下的显示效果
